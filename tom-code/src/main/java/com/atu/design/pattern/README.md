2. 饿汉式(静态代码块) 可用 Singleton2.java
3. 懒汉式(线程不安全) 不可用

5. 懒汉式(线程不安全，同步代码块) 不可用

优点：线程安全；延迟加载；效率较高

问题1：为什么要double-check？
 a、线程安全；
 b、性能问题，多线程访问不能及时响应

问题2：为什么要用volatile
 a、新建对象实际上有3个步骤（非原子操作）
 b、防止重排序带来的NPE（空指针）问题。

7. 静态内部类 推荐使用 Singleton7.java  线程安全 懒加载
8. 枚举 推荐使用————生产实践最佳写法 Singleton8.java 可以防止反序列化重新创建新的对象
